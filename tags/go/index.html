2017-13-10 ‐ Proxy Protocol: what is it and how to use it with Go
Today I became aware of the proxy protocol. The Proxy Protocol was designed to chain proxies / reverse-proxies without losing the client information. If you are proxying an HTTP(S) chances are that you have used the X-Forwarded-From header to keep the real remote address of the client making the request and not receving the proxy’s address instead. But this only works for HTTP(S): if you are proxying any other kind of TCP service, you are doomed.
